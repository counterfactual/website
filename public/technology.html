Technology

State channels are the foundational technology for useable distributed applications. They can be used in any interaction with a defined set of participants, such as payments or games.

A New Workflow for Dapps

For developers, off-chain applications follow a different pattern than dapp developers might be used to. Instead of prompting users to sign and submit transactions that get sent to a blockchain network directly, users sign (perhaps automatically) transactions that _could_ be sent to a blockchain, but are instead sent directly to their counterparties.

Counterfactual provides a simple client library, similar to Web3, that applications can use to communicate to wallets that implement the protocol.

A Protocol for State Channels

Counterfactual has defined a protocol for off chain applications that is maximally secure, flexible, and intuitive while also abstract enough to cover arbitrary blockchain transactions. At its core, the protocol introduces the notion of an `AppInstance` which can be installed, updated, and uninstalled within a state channel. Full specifications can be found on Github.

Built on the Ethereum Blockchain

All transactions that users sign while using the protocol are ultimately designed such that they could be placed on the blockchain in the case of a dispute. Counterfactual implements the minimum functionality needed for dealing with state channels of abstract transactions by combining the best in class research from teams like L4, Magmo, Sprites, Pisa, and others into a single on-chain set of contracts. To read more about the design, check out the contracts on Github.
